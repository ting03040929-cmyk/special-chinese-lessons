# 國文教材製作工具流指南 (Chinese Material Builder Guide)

本指南說明如何自動化製作與重組國中特教國文教材，支援語詞配圖準備、注釋投影片製作與 Word 課文/練習題自動排版組裝。

---

## 🛠️ 環境與套件需求 (Dependencies)
本工具流使用 Python 撰寫，執行前請確認已安裝以下套件：
- `python-docx`
- `python-pptx`
- `pymupdf` (fitz)
- `pillow`
- `pywin32` (Windows 環境下用於調用真實 PowerPoint 進行轉圖)

*本腳本支援 PEP 723 inline dependency metadata，使用 `uv run` 執行時將自動配置環境。*

---

## 🚀 完整教材製作流程 (Workflow Stages)

### 0. 第 0 階段：從普通班課文生成特教教材與 JSON 資料 (選填)
若備有普通班課文的純文字檔（例如 `夏夜課文.txt`），可指定金鑰一鍵自動呼叫 Gemini API 產生 `data.json` 與適合特教班的教材文字檔 `gemini_[課文名稱].docx`：
```powershell
uv run C:\Users\5A88\.gemini\config\skills\chinese-material-builder\scripts\material_builder.py generate-data --lesson-name 夏夜 --lesson-file 夏夜課文.txt --output-dir 7上_夏夜
```

### 0.5. 第 0.5 階段：利用 NotebookLM 製作特教手繪風課文簡報 (選填)
利用 Google NotebookLM 製作無注釋、課文敘述豐富的手繪風簡報：
1. **尋找或建立筆記本**：
   - 列出所有筆記本取得 ID：
     ```powershell
     nlm list notebooks
     ```
   - 或新建一個專屬筆記本：
     ```powershell
     nlm create notebook "114國中國文1上 L03 吃冰的滋味_特教簡報"
     ```
2. **上傳 Markdown 簡報大綱**：
   - 讀取第 0 階段產出的 `notebooklm_[課文名稱].md` 並上傳為來源：
     ```powershell
     $text = Get-Content -Raw -Encoding utf8 "路徑/notebooklm_[課文名稱].md"
     nlm add text "[筆記本ID]" $text --title "[課文名稱] 特教簡報大綱" --wait
     ```
3. **生成繁體中文簡報**：
   - 強制指定繁體中文（`zh-TW`）語系，避免預設生成英文版投影片：
     ```powershell
     nlm slides create "[筆記本ID]" --language zh-TW --confirm
     ```
4. **下載與提取簡報投影片**：
   - 將生成完畢的簡報下載為 PPTX 簡報檔：
     ```powershell
     nlm download slide-deck "[筆記本ID]" -f pptx -o "路徑/notebooklm_[課文名稱].pptx"
     ```

---

### 1. 第一階段：產生語詞配圖與簡報
1. 準備兩類語詞的配圖，並以語詞名稱命名為 `[語詞名稱].jpg` 或 `[語詞名稱].png` 統一放入 `[output-dir]/圖片/` 資料夾下：
   - **第一種：常見語詞 (12個)**：僅需要詞語配圖，**不需要產出 PPT 簡報**。
   - **第二種：難字語詞 (8個)**：需要詞語配圖，且**需要產出 PPT 簡報**（注釋投影片）。
2. 執行第一階段指令，系統會針對難字語詞產生 PPTX 簡報並自動完成投影片轉圖：
   ```powershell
   uv run C:\Users\5A88\.gemini\config\skills\chinese-material-builder\scripts\material_builder.py generate-assets --json-data data.json --output-dir [output-dir]
   ```
3. **人工檢核步驟**：指令執行完成後，教師必須人工確認 12 個常見語詞的配圖、8 個難字語詞的配圖以及難字 PPT 投影片轉圖。確認皆無問題後，才能進行下一步。

---

### 2. 第二階段：組裝最終 Word 檔
1. 在教師確認素材無誤後，執行組裝指令：
   ```powershell
   uv run C:\Users\5A88\.gemini\config\skills\chinese-material-builder\scripts\material_builder.py build-docx --template [範本Word路徑] --pdf [課文PDF路徑] --json-data data.json --assets-dir [output-dir] --output [最終Word輸出路徑]
   ```
2. 重組成功後，即可得到排版完美的 Word 檔。

#### 💡 第二階段 Word 組裝引擎之自動調整邏輯：
* **課文表格動態縮放**：組裝引擎會依據課文實際段落數（例如《吃冰的滋味》共 8 段），動態調整前三個課文表格（Table 0, Table 1, Table 2）的列數，自動刪除多餘的空白列（如 Table 2 縮減為 2 列並刪除原第 3 列），防止版面破碎。
* **朗讀課文精簡處理**：第 24 頁的朗讀課文區塊不再全文合併，而是僅擷取「第一段」與「中間段落」的課文填入朗讀框中，避免特教學生閱讀過長文章。
* **連連看答案隨機打散**：第 23 頁的連連看配對中，常見語詞的對應圖片（`image21` ~ `image30`）會經過重新打散對調，確保正確答案圖卡不會剛好落在語詞的正下方，符合「連連看」的題型需求。
* **注釋投影片編號**：在 Page 11 至 Page 14 的 8 個注釋投影片圖片上方，自動加上 `1.` 到 `8.` 的粗體題號段落（字型微軟正黑體 14pt，置左），便於指引學生。
* **檔案衝突與防覆蓋機制**：
  - 若目標檔案（例如 `v8.docx`）目前正被 Word 開啟鎖定，寫入時會因 `PermissionError` 自動以版本號遞增另存（如 `v9.docx`、`v8_新.docx`）。
  - **重要原則**：不可強行覆蓋或刪除舊的版本檔案。若需要將修正版存回指定版本（如 `v8`），應先將舊版手動更名為 `v8_舊.docx`，再將新產出的檔案更名為 `v8.docx`，確保手動編輯歷史不被覆蓋。

## ⚠️ 常見錯誤與排障 (Common Mistakes)
1. **圖片名稱與語詞名稱不一致**：放入 `圖片/` 的圖檔名稱，必須與 JSON 中的 `word` 欄位字串 100% 相同（包含副檔名大小寫，如 `蜜糖.jpg`）。
2. **Word 檔案佔用鎖定**：若執行 `build-docx` 時目標 Word 檔已被 Word 開啟，會丟出 PermissionError 並自動將輸出存為新檔名。
3. **PowerPoint 未安裝**：若執行環境沒有 PowerPoint，第一階段 PPTX 轉圖會失敗。此時需要手動以 PowerPoint 開啟產生出的 PPTX，另存為圖片至 `slide_images/` 目錄。
